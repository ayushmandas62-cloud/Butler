# Butler Coding Standard

## 1. One Responsibility

Each script should do only one job.

Example:
- doctor.sh → diagnostics
- start.sh → startup
- stop.sh → shutdown

## 2. No duplicated code

If code is needed twice, move it into common.sh.

## 3. Always check errors

Never assume a command succeeds.

## 4. Every feature must be testable.

## 5. Every feature gets its own Git commit.

## 6. Every public function must have a comment explaining what it does.
